# FoodRush Backend

Ce dossier est reserve au futur backend/API de FoodRush.

Structure prevue:

- `api/` pour les routes et controllers
- `services/` pour la logique metier
- `models/` pour les schemas et types de donnees
- `config/` pour la configuration

Le frontend continue pour l'instant d'utiliser `src/services/api.ts` comme point d'entree API.
